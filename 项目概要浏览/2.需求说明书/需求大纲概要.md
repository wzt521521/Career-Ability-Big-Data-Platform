# 职业能力大数据服务平台

# 项目开发大纲（需求分析概要）

---

# 第一章 项目概述

## 1.1 项目名称

职业能力大数据服务平台

---

## 1.2 项目背景

随着互联网招聘平台的快速发展，大量招聘信息实时产生，其中包含职位名称、薪资水平、学历要求、工作经验、岗位技能、地区分布、企业规模等丰富的数据资源。这些数据能够真实反映当前就业市场的发展状况及企业人才需求趋势。

目前，高校在专业建设、课程改革及就业指导过程中，主要依赖企业调研、毕业生就业统计等传统方式获取就业信息，数据更新周期较长，覆盖范围有限，难以及时反映市场变化。同时，学生在职业规划和岗位选择过程中，也缺乏客观、系统的就业数据支撑。

因此，本项目拟建设"职业能力大数据服务平台"，利用大数据采集、存储、计算及分析技术，对互联网公开招聘信息进行持续采集和分析，形成行业就业分析报告、岗位能力分析、技能需求分析及就业趋势预测，为高校教学改革、专业建设及学生职业规划提供数据支持。

---

## 1.3 项目目标

本项目以就业大数据分析为核心，通过构建完整的数据处理流程，实现以下目标：

1. 建立稳定、高效的招聘信息采集平台，实现多招聘网站数据的自动采集。
2. 建立统一的数据清洗与治理流程，提高数据质量和可用性。
3. 构建就业数据仓库，为后续统计分析提供数据基础。
4. 对岗位数量、薪资水平、学历要求、技能需求、行业发展趋势等维度进行统计分析。
5. 自动生成就业分析报告，为高校管理部门提供辅助决策依据。
6. 根据学生专业、技能及就业意向推荐匹配岗位，提高就业指导质量。
7. 提供开放API接口，为第三方系统提供数据共享能力。

# 第二章 系统总体设计

## 2.1 建设目标

系统采用"数据采集—数据处理—数据分析—数据展示—辅助决策"的总体架构，构建覆盖就业数据全生命周期管理的大数据分析平台。

整体建设目标包括：

* 数据自动采集
* 数据统一存储
* 数据智能分析
* 数据可视化展示
* 数据开放共享

---

## 2.2 总体架构

平台采用分层架构设计，共分为六个层次：

### （1）数据源层

负责互联网招聘数据采集。

数据来源包括：

* BOSS直聘、智联招聘、前程无忧、拉勾网、猎聘网等
* 公开的招聘数据集（Kaggle、天池等）
* 政府公开的就业统计数据

> **说明：** 项目优先使用公开数据集和本地模拟数据进行开发和演示。针对真实招聘网站的爬虫程序作为技术验证，实际运行时遵守 robots.txt 协议，仅用于学习和研究目的。

采集内容包括：

* 企业信息（名称、规模、行业、性质）
* 岗位信息（名称、薪资、地点、经验要求）
* 技能要求
* 学历要求
* 福利待遇
* 发布时间

---

### （2）数据采集层

负责互联网数据采集与预处理。

主要功能：

* 分布式爬虫调度
* 定时采集
* 增量采集
* 数据去重
* 失败重试
* Kafka消息推送

---

### （3）数据存储层

根据数据类型采用不同存储方式。

| 数据库   | 用途         | 说明               |
| -------- | ------------ | ------------------ |
| MySQL    | 业务数据与分析结果 | 核心存储，必须部署     |
| Redis    | 热点缓存     | 核心组件，必须部署     |
| HDFS     | 原始数据归档 | 大数据组件，Docker部署 |
| Hive     | 离线分析仓库 | 大数据组件，Docker部署 |
| HBase    | 实时查询     | 大数据组件，按需部署   |

---

### （4）数据分析层

基于Spark完成离线批处理分析，基于MySQL完成日常统计查询。

分析内容包括：

* 岗位统计（数量、热度、增长率）
* 行业统计（行业分布、趋势）
* 薪资分析（均值、区间、排行）
* 技能分析（词频、组合、趋势）
* 学历分析（分布、交叉对比）
* 城市分析（排名、薪资对比）
* 企业分析（规模、活跃度）
* 趋势分析（日增量、月度变化）

---

### （5）业务服务层

提供后台业务功能：

* 用户管理（注册、登录、角色与权限控制）
* 数据分析（多维度统计查询与可视化数据接口）
* 报告生成（自动汇总分析结果，生成PDF/Word/Excel报告）
* 岗位推荐（根据学生画像匹配岗位，返回推荐列表）
* API服务（RESTful接口，Token认证）

---

### （6）展示层

采用Web可视化展示分析结果。

主要包括：

* 数据大屏（就业概览仪表盘）
* 图表分析（ECharts可视化图表）
* 报告中心（在线预览与下载）
* 岗位推荐（个性化推荐列表）
* 系统管理（用户、角色、日志管理）

---

# 第三章 功能模块设计

系统划分为六个核心功能模块。

## 3.1 用户与权限管理模块

负责平台用户身份认证与权限控制。

主要功能：

* 登录认证（账号密码 + Token）
* 用户管理（增删改查、批量导入）
* 角色管理（创建与分配）
* 权限分配（页面级 + 数据级）
* 操作日志（关键操作审计）
* 密码修改与重置

角色定义：

| 角色       | 主要权限                         |
| ---------- | -------------------------------- |
| 系统管理员 | 全部功能，含用户管理和系统配置   |
| 数据分析员 | 数据采集管理、分析、报告生成、API |
| 教师       | 查看分析结果、下载报告、查看推荐 |
| 学院管理员 | 查看本院数据、下载报告           |
| 学生       | 查看岗位推荐、浏览部分分析数据   |

---

## 3.2 分布式数据采集模块

负责互联网招聘信息的自动化采集。

主要功能：

* 采集源配置（URL模板、翻页规则、采集频率）
* 采集任务管理（创建、启停、调度）
* 定时采集（基于Cron表达式）
* 增量采集（基于时间戳和唯一标识去重）
* 数据去重（URL哈希 + 职位ID双重校验）
* Kafka消息发送（采集结果实时推送）
* 失败重试（最多3次，指数退避）
* 日志监控（采集量统计、异常告警）

**采集流程：**

```text
采集任务启动 → 加载URL模板 → 请求页面 → 解析HTML
    → 数据去重 → 格式标准化 → 发送Kafka → 记录日志
```

**输出：** 标准化岗位原始数据（JSON格式）。

---

## 3.3 数据分析模块

负责对招聘数据进行多维度统计分析。

### （1）岗位分析

* 岗位总量及分类统计
* 热门岗位 TOP 20
* 岗位月度增长率

### （2）薪资分析

* 各岗位平均薪资、中位数
* 薪资区间分布
* 高薪岗位 TOP 20
* 各城市薪资对比

### （3）学历分析

* 学历要求比例
* 各学历对应岗位数量
* 学历与薪资交叉分析

### （4）技能分析

* 热门技能词频 TOP 30
* 技能关联度分析（技能组合出现频率）
* 技能需求月度变化趋势

### （5）地域分析

* 城市岗位数量排名
* 热门就业城市 TOP 10
* 城市平均薪资排名

### （6）企业分析

* 企业规模分布
* 企业行业分类
* 企业招聘活跃度排名

### （7）趋势分析

* 每日新增岗位变化
* 月度/季度变化趋势
* 行业发展趋势（同比、环比）

---

## 3.4 报告自动生成模块

根据分析结果自动生成就业分析报告。

支持格式：PDF、Word、Excel

报告内容：

* 行业概况
* 岗位统计
* 薪资分析
* 技能需求分析
* 学历与经验分析
* 地域分析
* 趋势预测
* 教学建议

功能支持：

* 一键生成与导出
* 报告模板管理
* 历史报告归档与检索

---

## 3.5 岗位推荐模块

根据学生信息与岗位要求的匹配程度，推荐合适的就业岗位。

推荐依据：

* 所学专业
* 已掌握技能
* 学历层次
* 意向城市
* 期望薪资

推荐算法采用**基于内容的匹配**：将学生画像（专业、技能、学历、意向城市、期望薪资）与岗位画像（要求技能、学历、城市、薪资范围）进行多维度加权匹配，技能匹配权重最高（约40%），按综合得分降序排列返回 TOP 20。

推荐展示内容：

* 岗位名称、企业名称
* 岗位薪资范围
* 技能匹配度百分比
* 匹配/未匹配技能对比
* 推荐指数

---

## 3.6 开放API模块

为第三方系统提供统一的数据访问接口。

主要接口：

* 岗位统计接口（岗位数量、分类、排行、趋势）
* 薪资分析接口（平均薪资、区间分布、城市对比）
* 技能分析接口（热门技能、趋势、关联分析）
* 地域分析接口（城市排名、薪资对比、热力图）
* 报告查询接口（报告列表、详情、下载）
* 推荐结果接口（个性化推荐、画像更新）

接口规范：

* 采用 RESTful 风格
* 返回 JSON 数据
* 基于 Token 进行身份认证
* 支持分页与时间范围筛选参数

# 第四章 技术架构

采用前后端分离架构，结合大数据处理平台。

**前端层**

* Vue 3 + Element Plus
* ECharts（数据可视化）
* Axios（HTTP请求）

**业务层**

* Spring Boot 2.7+
* Spring MVC
* Spring Data JPA
* Redis（缓存）
* Kafka（消息队列）

**数据分析层**

* Spark 3.x / Spark SQL（离线批处理）
* MapReduce（基础数据处理）
* Hive（数据仓库查询）
* HBase（实时查询）

**数据存储层**

* MySQL 8.0（业务数据、分析结果）
* HDFS（原始数据归档）
* Redis 7（热点缓存、Session）

**部署层**

* Linux（CentOS 7+ / Ubuntu 20.04+）
* Docker + Docker Compose
* Nginx（反向代理）

# 第五章 系统业务流程

系统总体业务流程如下：

```text
招聘网站数据 / 公开数据集
        │
        ▼
分布式数据采集（爬虫 + Kafka）
        │
        ▼
Kafka 消息队列
        │
        ▼
Spark 数据清洗与ETL
        │
        ▼
HDFS / Hive 数据仓库
        │
        ▼
Spark SQL / Hive 统计分析
        │
        ▼
MySQL 分析结果存储（Redis缓存）
        │
        │
   ┌────┴────────┐
   ▼              ▼
就业分析报告    数据可视化大屏
   │              │
   ├──────────────┤
   ▼              ▼
岗位推荐      开放API服务
```

# 第六章 用户使用场景

## 场景一：教师查看就业趋势并调整课程

教师登录系统，查看"Java开发"岗位的技能需求排行，发现企业对 Spring Boot（92%）、Redis（83%）、Docker（61%）的需求远高于学校当前教授的 JSP、Servlet。教师下载就业分析报告作为课程改革依据，提交教务处审核。

## 场景二：学生获取个性化岗位推荐

学生登录后完善个人画像（专业：计算机科学，技能：Java/MySQL，意向城市：上海/杭州，期望薪资：8K-15K）。系统计算匹配度后返回推荐岗位列表。学生发现目标岗位普遍要求 Docker 和 K8s，决定优先学习这两项技能。

## 场景三：管理员生成月度就业分析报告

管理员设置统计时间范围，选择"月度就业分析报告"模板，系统自动汇总岗位统计、薪资分析、技能趋势、地域分布等数据，生成 PDF 报告。管理员预览确认后下载并分发至各学院。
